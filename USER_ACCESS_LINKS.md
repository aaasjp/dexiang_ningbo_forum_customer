## 三、Session 参数说明

所有链接的 session 参数格式为：
```
appid={staff_id},name={姓名},depatment=人力资源部,orgId=2,jobTitle={角色}, gender={性别}, status=1,jobNo={staff_code}
```

### 参数说明：
- `appid`: 员工ID（staff_id）
- `name`: 员工姓名
- `depatment`: 部门名称（固定为"人力资源部"）
- `orgId`: 组织ID（固定为 2）
- `jobTitle`: 角色类型
  - 超级管理员：超级管理员
  - 行长：行长
  - 部门管理员：部门管理员
  - 其他：普通人员
- `gender`: 性别（1=女，2=男）
- `status`: 状态（固定为 1）
- `jobNo`: 员工工号（staff_code）

### 注意事项：
1. 所有 session 参数已经过 `encodeURIComponent` 编码处理
2. 系统会在页面首次加载时自动从 URL 中读取 session 参数
3. 如果 URL 中有 session 参数，会优先使用 URL 中的值，并重新获取用户信息
4. 用户端主要用于论坛浏览和互动
5. 管理端主要用于内容管理、用户管理等后台操作
